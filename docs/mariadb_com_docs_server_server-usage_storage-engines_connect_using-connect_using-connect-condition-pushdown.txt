Using CONNECT - Condition Pushdown
The CONNECT storage engine.
The
ODBC
,
JDBC
,
MYSQL
,
TBL
and WMI table types use engine condition pushdown in order to restrict the number of rows returned by the RDBS source or the WMI component.
The CONDITION_PUSHDOWN argument used in old versions of CONNECT is no longer needed because CONNECT uses condition pushdown unconditionally.
