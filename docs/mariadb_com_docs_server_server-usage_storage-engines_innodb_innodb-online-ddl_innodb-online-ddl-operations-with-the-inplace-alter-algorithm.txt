InnoDB Online DDL Operations with the INPLACE Alter Algorithm
Learn about operations supported by the INPLACE algorithm, which rebuilds the table but allows concurrent DML, offering a balance between performance and availability.
Supported Operations by Inheritance
When the
ALGORITHM
clause is set to
INPLACE
, the supported operations are a superset of the operations that are supported when the
ALGORITHM
clause is set to
NOCOPY
. Similarly, when the
ALGORITHM
clause is set to
NOCOPY
, the supported operations are a superset of the operations that are supported when the
ALGORITHM
clause is set to
INSTANT
.
Therefore, when the
ALGORITHM
clause is set to
INPLACE
, some operations are supported by inheritance. See the following additional pages for more information about these supported operations:
InnoDB Online DDL Operations with ALGORITHM=NOCOPY
InnoDB Online DDL Operations with ALGORITHM=INSTANT
