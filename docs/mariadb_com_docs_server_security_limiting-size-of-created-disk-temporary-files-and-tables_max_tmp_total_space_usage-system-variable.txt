max_tmp_total_space_usage System Variable
Reference for the global system variable that defines the maximum cumulative disk space all user connections can consume for temporary files and tables.
max_tmp_total_space_usage
Description: The maximum total size in bytes of all temporary file and temporary table usage over all connections. A value of 0 disables this feature. Set in blocks of 65536, rounded down if not a multiple of 65536. See
Limiting Size of Created Disk Temporary Files and Tables Overview
for details. Called max_total_tmp_space_usage in
MariaDB 11.5.0
only.
Commandline:
--max-tmp-total-space-usage=num
Scope: Global
Dynamic: Yes
Data Type:
numeric
(bigint unsigned)
Default Value:
1099511627776
Range:
0
to
18446744073709551615
(blocks of
65536
)
Introduced:
MariaDB 11.5
