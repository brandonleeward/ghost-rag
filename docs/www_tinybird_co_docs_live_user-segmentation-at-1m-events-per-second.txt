User segmentation at 1M events per second
Game developers generate revenue through advertisements and player purchases. Realtime user segmentation can help increase revenue by serving relevant offers, but this can be challenging with millions of concurrent users. Join us as we demonstrate effective user segmentation at over 1M player events per second.
In this livestream, we'll demonstrate:
The importance of realtime architectures for handling high user concurrency in user segmentation
How to send 1M rows per second from an application to a Tinybird data source
How to create SQL Pipes to analyze user behavior and assess whether to serve ads or purchase offers
How to publish SQL Pipes as API endpoints using dynamic query parameters
