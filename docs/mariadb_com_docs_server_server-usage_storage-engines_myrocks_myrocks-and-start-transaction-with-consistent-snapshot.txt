MyRocks START TRANSACTION WITH CONSISTENT SNAPSHOT
MyRocks storage engine transactions with consistent snapshot.
FB/MySQL has added new syntax:
Copy
START TRANSACTION
WITH
CONSISTENT ROCKSDB|INNODB
SNAPSHOT
;
The statement returns the binlog coordinates pointing at the snapshot.
MariaDB (and Percona Server) support extension to the regular
Copy
START TRANSACTION
WITH
CONSISTENT
SNAPSHOT
;
syntax as documented in
Enhancements for START TRANSACTION WITH CONSISTENT SNAPSHOT
.
After issuing the statement, one can examine the
binlog_snapshot_file
and
binlog_snapshot_position
status variables to see the binlog position that corresponds to the snapshot.
